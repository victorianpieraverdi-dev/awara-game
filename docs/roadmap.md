---

## Принцип

- Каждая фаза заканчивается **playable build** на `https://awara-v258-modular-epfubmxp.devinapps.com` или преемнике.
- **Вертикальные срезы**, не горизонтальные слои: лучше один полностью рабочий feature, чем 5 заготовок.
- **Канон не двигаем без необходимости.** Если чего-то нет в Master Codex / GDD / видении — это импровизация, требует подтверждения.
- **Минимум зависимостей вначале.** Все внешние API подключаются после фазы 2.
- **localStorage-first**, Firestore/backend — после MVP.

## Phase 5 — Космос игрока + Карта мироздания (8–12 недель)

**Цель:** уровни 3 и 4 видения.

Перед началом этой фазы нужно не продолжать сжимать текущую desktop-вселенную в телефон, а переложить её на новую архитектуру:

- `UniverseState` — уровень игрока, свет, сферы, карты, связи, храмы, пространства;
- `UniverseProgression` — лестница Ахамкара → Душа → Джива → Дух → Собор → Настольная игра;
- `UniverseDesktopView` — полноценная карта для большого экрана;
- `UniverseMobileFocusView` — лаконичный фокус-экран для телефона;
- `UniverseMobileMapView` — обзорная карта с жестами;
- `UniverseBoardView` — будущий верхний слой настольной игры.

Ключевой UX-принцип:

```text
чем глубже архитектура игрока внутри, тем проще и чище интерфейс снаружи
```

| # | Блок | Часы |

## Phase 6 — Светообмен + Свои игры (long-term)

Пассивный доход (накопленные ключи и карты), служение всем живым существам, мета-уровень "своих игр" — игрок начинает создавать собственные миры.

Это отдельный архитектурный этап, требующий backend, multi-tenancy, экосистемы пользовательского контента. **6+ месяцев** работы.
