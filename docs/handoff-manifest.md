## 0. TL;DR — 30-секундный брифинг

**AWARA** — это не "игра", а **сакральная операционная система**, рассчитанная на 30 000 лет. Её задача — провести игрока (и через него Землю-Логос) от инертности через осознанность к выходу за пределы Брахманды.

- **Visionary:** Pavel Burdinov (PBerty, `burdinovpaveld@gmail.com`), русскоязычный, 3+ года ведёт этот проект.
- **Канон:** 21 Космический Агент × 33 Матрицы культур × 14 Лок (плотностей) × 9 чакр-мер. Всё описано в 53 лор-файлах.
- **Текущий код:** статический мульти-страничный prototype на чистом HTML/CSS/JS, локальный сервер `python -m http.server 8765`, deploy на `https://awara-v258-modular-epfubmxp.devinapps.com`.
- **Состояние:** дизайн-фундамент (4 экрана) собран. Дальше — фундамент данных (JSON), daily loop, карты, натальная, AI-Оракул, генерация картинок и музыки.
- **Стиль работы Pavel'а:** последовательно (A→B→C→D→E), терсё, по-русски, **не пушить в GitHub** ("оставь у меня локально"), mobile-first, всегда деплой и скриншоты.

## 3. Видение AWARA — полная картина

### 3.1 Главная сверх-цель

Игра — инструмент **подготовки игрока и планетарного Логоса** к переходу на следующий вибрационный уровень. Рассчитана на **30 000 лет**, фрактальная, **неотделима от жизни**. Финал: игрок выходит из игры с интегрированными принципами и **создаёт свою**.

### 3.2 6 уровней эволюции игрока (горизонтальная ось)

| # | Уровень | Содержание |
|---|---------|------------|
| 1 | **Инициация / Душа** | Текущий контур (ДУША · ОСНОВА · СЕРДЦЕ · МЕЧТА). 4 корневые сферы + подсферы. Прокачивается личный состав. |
| 2 | **Земля игрока** | Васту-храм, дом, личное окружение, стихии (земля/вода/огонь/воздух/эфир). 9 зон + центр. |
| 3 | **Космос игрока** | Звёздный храм, связи с другими игроками, NPC, синхронизации с космосом. |
| 4 | **Карта мироздания / божественные игры** | Игрок становится божественным игроком. Интерактивная карта Земли + 62/64 фестивальных точек на планетарном Логосе. |
| 5 | **Светообмен** | Отдача всем живым существам. Пассивный доход, служение. |
| 6 | **Свои игры** | Трансценденция. Разворачивается через **Матрицу 33 (Адвайта-Сиддха)** — Пранава-деха либо Парамукти. Игрок либо выходит полностью, либо возвращается как Дживанмукта-инструктор. |

**Принцип:** каждый уровень включён в следующий. Прокачивается душа → окружение → земля → космос → божественная игра.

### 5.1 Матрица 33 — особый статус

Это **не очередная карта**, а **мета-матрица выхода**. Смысл:

- 32 матрицы = разные интерфейсы Источника
- 33-я = инструкция по **демонтажу всех интерфейсов** и выходу из Брахманды

**Финальная формула М33:**
```
Пашу (Обусловленность) → Вира (Тапас) → Сушумна → Пробой Трёх Грантхи →
Лайя (Остановка Ума) → Маха-деха (Космическое Тело) → Дживанмукти (Освобождение при жизни) →
Пранава-деха (Тело Света) → Парамукти (Выход из Брахманды)
```

**Игровые следствия М33:**
- Не выбирается как стартовая.
- Открывается только после прохождения значительной части пути.
- Не даёт обычные карты — даёт **разрушение** карт (сжигание самскар, разрыв грантхи, растворение коллекции в Свете).
- Финальная сцена игры = краш-тест: игрок добровольно удаляет свой код из Брахманды.
- Артефакты М33 не накапливаются, а **используются для растворения других артефактов**:
  - Меч Рудры рубит привязки
  - Чаша Тишины выпивает шум
  - Сломанное Колесо Кармы стирает счётчики
  - Кристалл Алхимической Плоти переплавляет тело
- Это **анти-прогрессия**: на низких уровнях накапливаешь, на М33 разучиваешься накапливать.

**6 фракций М33:**
1. Спящие (Пашу)
2. Герои (Виры)
3. Алхимики Сушумны
4. Свидетели (Сакши)
5. Дживанмукты (Дивья)
6. Световые Архитекторы

## 7. Светкоин-экономика по уровням

Принцип: **чем БЛИЖЕ к Источнику, тем БОЛЬШЕ света за единицу опыта** (нулей **меньше**, не больше).

| Уровень | Прирост за событие |
|---------|---------------------|
| Инициация | ~`0.0000000001 ✦` (10⁻¹⁰) |
| Земля игрока | ~`0.000001 ✦` (10⁻⁶) |
| Космос игрока | ~`0.0001 ✦` (10⁻⁴) → 0.001 → 1 → 10... |
| Игра Мироздания | `0.01–0.1 ✦`, в т.ч. **пассивно** через накопленные ключи/карты |
| Светообмен | `1–10 ✦` пассивно |
| Свои игры | `∞` (за пределами счёта) |

Светкоин-банк / дашборд: видно вклад каждого игрока, топ контрибьюторов.

**Технически:** хранить балансы как `BigInt` или `string`-представление, чтобы не терять precision на 10⁻¹⁰.

## 8. Тигель (Crucible) — сердце ежедневной практики

### 8.1 Что есть

Сейчас в `tigel.html` 3 вкладки (log / balance / alchemy). Есть базовая У-Син механика → светкоин.

### 8.2 Что должно быть (видение Pavel'а)

**На вход:** игрок пишет лог дня (свободный текст, в идеале минут 5–15 рефлексии).

**Распределение:** AI-агент разносит содержимое по:
- Сферам инициации (ДУША · ОСНОВА · СЕРДЦЕ · МЕЧТА)
- Зонам Земли игрока (9 Васту-зон)
- Космосу игрока (если разблокирован)
- Картам (если затронута тема конкретного агента/матрицы)

**На выход — 7 параметров:**
1. **Энергия дня** (по матрице восприятия)
2. **Энергия натальной карты** (по натальной)
3. **Итоговая энергия + гармонизация**
4. **Что будет завтра** (forecast)
5. **Образ дня** (сгенерированный визуал)
6. **Музыка** (Suno / процедурно)
7. **Айурведический совет** (питание, поведение)

**Дополнительные элементы:**
- **Двунаправленность:** длинный текст в Тигле распределяется по сферам и обратно (можно достать из сферы и обогатить).
- **Telegram-бот клиент:** написал в бот → запись попала в Тигель, не заходя в игру.
- **Искра** в пространстве — летает по сферам, собирает информацию, даёт подсказки. **Открытый вопрос**: оставлять или нет.
- **Оракул** — диалог о следующем уровне, как сгармонизировать текущие.

## 14. Code conventions

### 14.1 Общие правила

- **Минимальные правки.** Не делать массовый рефакторинг без явной просьбы.
- **Нет комментариев-объяснений к диффу.** Комментарий "// исправлено для..." недопустим. Комментарий должен описывать **код в общем**, а не твою правку.
- **Нет эмодзи в коде, файлах, PR-описаниях** (если Pavel не попросил).
- **Существующие конвенции > свои предпочтения.** Смотреть на соседние файлы.
- **Imports сверху файла.** Никогда не импортировать внутри функций.
- **Безопасность:** не логировать секреты, не коммитить .env и подобное.

### 14.2 HTML/CSS/JS специфика этого проекта

- Каждый экран = свой `.html` файл с inline `<style>` или подключением `css/main.css`.
- JS — модули в `js/`, подключаются в HTML через `<script type="module">` или обычный `<script>`.
- localStorage — основа state. Прежде чем перейти на бекэнд, всё должно работать на чистом frontend.
- Анимации — CSS-keyframes + JS на ивентах. Не добавлять heavy фреймворки (нет React/Vue/Svelte) — Pavel хочет легковесность.

### 14.3 Git rules

- **НЕ пушить в GitHub без явной просьбы Pavel'а.** ("оставь у меня локально")
- Коммиты делаются локально для версионирования, но без push.
- Не amend, не force push, не reset --hard.
- Не делать `git add .` — добавлять файлы выборочно.
