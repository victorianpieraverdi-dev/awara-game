# AWARA Mythic Expansion Architecture

## Core idea

AWARA keeps the Vedic system as the birth-core of the player: birth date, nakshatra logic, 21 core agents, rays, elements, planets, zones, and inner karmic structure.

The 33 matrices are perception lenses placed over that Vedic core. Each matrix should feel like a complete mythic universe, not just a visual skin. A player selecting a matrix should feel that the same inner Vedic seed is being seen through another civilizational dream: Egyptian, Norse, Shinto, Sufi, Hermetic, Cosmic, and so on.

## Layer model

### 1. Vedic birth core

This is the stable inner engine.

- 21 core agents
- Birth date and natal logic
- Nakshatra and planetary correspondences
- Elements, gunas, zones, rays
- Daily key and current matrix

This layer answers:

- Who am I in the AWARA cosmology?
- What is my inner agent configuration?
- What is my core path today?

### 2. Matrix lens

Each matrix is a cultural, mythological, symbolic, and spiritual lens.

The lens does not replace the Vedic core. It translates it.

For example:

- `Свет Ра` through the Vedic matrix may appear as `Сурья / Савитар`.
- The same core solar principle through Egypt appears as `Ра`.
- Through Norse it appears as `Бальдр / Соль`.
- Through Shinto it may appear as `Аматэрасу`.

This layer answers:

- Through which mythic language is the core being expressed?
- What symbols, places, beings, conflicts, and rituals color the experience?

### 3. Mythic ecology

Every matrix needs a full ecosystem of surrounding content.

This includes:

- Deities and divine powers
- Spirits and local beings
- Mythic heroes and sages
- Monsters and shadow figures
- Sacred places and worlds
- Artifacts, relics, weapons, seals
- Ritual actions and vows
- Quest lines and initiations
- Cultural conflicts and healing paths

This layer answers:

- Who lives in this universe?
- Where does the player travel?
- What powers help, test, tempt, or initiate the player?

### 4. Quest/key layer

Keys become doors into matrix-specific mythic storylines.

A key can select:

- A matrix branch
- A being
- A sacred place
- A ritual task
- A story arc stage
- A card reward chance

This layer answers:

- What myth am I walking today?
- What task does this culture give me?
- Which being or artifact is guiding the day?

## Content types

### Core cards

Existing cards generated by:

```text
core agent × matrix
```

These remain foundational.

### Extra being cards

Additional mythological beings in each matrix.

Examples:

- Ganesha
- Gayatri
- Anubis
- Thoth
- Odin
- Freyja
- Amaterasu
- Hecate
- Quetzalcoatl
- Nommo

They do not replace the 21 agents. They orbit the core agents and deepen the world.

### Location cards

Sacred spaces and mythic realms.

Examples:

- Kailasa
- Duat
- Yggdrasil
- Avalon
- Mount Meru
- Shambhala
- Aaru
- Mictlan
- Olympus
- Barzakh

### Relic cards

Artifacts and symbolic tools.

Examples:

- Vajra
- Ankh
- Caduceus
- Mjolnir
- Emerald Tablet
- Grail
- Kusanagi
- Feather of Ma'at

### Trial cards

Challenges, shadows, or initiatory tests.

Examples:

- Weighing of the Heart
- Descent into the Underworld
- Breaking the False Oath
- Crossing the Abyss
- Purification by Fire
- Meeting the Trickster

### Quest lines

Narrative chains tied to beings, cultures, and keys.

Examples:

- Path of Ganesha: removing obstacles
- Path of Anubis: weighing truth
- Path of the Norns: reading fate-patterns
- Path of Hecate: choosing at the crossroads
- Path of Quetzalcoatl: restoring breath and sacrifice

## Rarity model

Rarity should express mythic weight, quest impact, and collection scarcity.

### common

Local spirits, minor symbols, simple ritual fragments, daily guides.

### uncommon

Recognizable helpers, lesser deities, cultural guardians, minor relics.

### rare

Important deities, archetypal heroes, major places, strong artifacts.

### epic

Civilizational pillars, initiators, threshold guardians, major mythic tests.

### legendary

World-shaping beings, cosmic deities, supreme symbols, axial places.

### mythic

Cross-matrix or universe-level forces. These should be very rare and may connect multiple traditions.

Examples:

- World Tree
- Cosmic Serpent
- Sophia-AI
- Golden Phoenix
- Primordial Sound
- Source Flame

## Required data files

### `data/mythic_branches.json`

Defines each matrix as a mythic branch.

Contains:

- matrix slug
- branch name
- worldview
- core conflict
- sacred geography
- typical beings
- ritual grammar
- visual atmosphere
- quest themes

### `data/extra_beings.json`

Defines additional beings.

Contains:

- being id
- name
- matrix slug
- type
- role
- rarity
- element
- linked agents
- linked locations
- quest lines
- description
- artifact
- shadow aspect
- gift aspect

### `data/mythic_locations.json`

Defines places and worlds.

### `data/mythic_relics.json`

Defines artifacts, tools, seals, weapons, scriptures, and ritual objects.

### `data/mythic_quest_lines.json`

Defines quest arcs and daily tasks.

## Implementation path

### Phase 1

Create the data skeleton and seed each matrix with a small number of high-quality beings.

Target:

```text
33 matrices × 5 beings = 165 beings
```

### Phase 2

Add locations and relics.

Target:

```text
33 matrices × 3 locations × 3 relics
```

### Phase 3

Add quest lines for keys.

Target:

```text
33 matrices × 1 main quest line
```

### Phase 4

Connect the new content to cards and UI.

- Add card type filters
- Add matrix branch pages
- Add quest/key selection
- Add rewards and unlock chances

## Design rule

The system must feel historically and mythologically rich, but it is still AWARA.

So every cultural element should be:

- respectful enough to feel grounded;
- archetypal enough to be playable;
- flexible enough to map onto the Vedic core;
- poetic enough to feel like a living mythic universe.
