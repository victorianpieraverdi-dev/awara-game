## 4. Технический стек (фиксированный)

- **Чистый HTML/CSS/JS.** No React/Vue/Svelte/Angular. No bundlers. No TypeScript (можно JSDoc).
- **Один файл = один экран** (`index.html`, `dashboard.html`, `tigel.html`, `matrices.html`, `earth-player.html`, `initiation-space.html`).
- **Inline `<style>`** допустим. Для общего — `css/main.css` (если появится).
- **Скрипты** в `js/`. Без модульных бандлеров — `<script type="module">` разрешён.
- **Состояние** — `localStorage`, ключ `awara_v255_state` (НЕ `awara_initiation_v1` как в манифесте; манифест устарел в этом месте).
- **Бэкенда нет.** Phase 0-2. Появится в Phase 3 (Оракул).

## 6. Светкоин

- Хранится в `state.totalLight` как **сырое число (raw)**.
- Отображается как `(raw / 1e10).toFixed(10)` → формат `0.0000004250` (10 знаков после точки).
- Множитель множится на уровень эволюции игрока (vision §2).
- Ранги Державы РА (`js/core.js` STAGES): 0 / 3000 / 7000 / 10000 / 25000 / 50000 → ИНИЦИАТ → ВОИН СВЕТА → МУДРЕЦ → ЦАРЬ → БУДДА → ПЛАНЕТАРНЫЙ ЛОГОС.

## 6.5. Секреты — НИКОГДА не в коде

- API-ключи (OpenRouter, Anthropic, OpenAI, Mapbox, любые) **никогда** не вписываются в `.html`/`.js`-файлы напрямую.
- Паттерн для пользовательских ключей: чтение из `localStorage`, при отсутствии — `prompt()` пользователю, сохраняем результат в `localStorage`. Ключ хранится только в браузере пользователя, никогда не уезжает в репо или deploy URL. Пример: см. функцию алхимии в `tigel.html`.
- Если в коде нашёлся hardcoded ключ — это критический баг. Немедленно: (а) сообщить Pavel'у, чтобы он отозвал ключ у провайдера, (б) заменить на паттерн localStorage+prompt, (в) если ключ уже попал в git history — пересобрать ветку через orphan, чтобы старые коммиты не было в публичном репо.
- Серверные ключи (Phase 3+ когда появится бекэнд) — через переменные окружения или secrets-менеджер платформы. **Не** через `localStorage`, **не** через коммиты.
- При деплое на публичный URL: помнить что весь JS виден в DevTools. Любой ключ в коде = ключ в публике.

## 7. Git-правила

- **Push разрешён** только в shared remote (этот репо). Цель — синхронизация между аккаунтами.
- **НЕ** force push в master. Свои feature-ветки — `--force-with-lease` ок.
- **НЕ** `git add .` — добавлять только конкретные файлы.
- **НЕ** amend, **НЕ** reset --hard.
- **НЕ** пропускать pre-commit hooks (`--no-verify`).
- **Один коммит = одна T-XXX задача.** Сообщение коммита на русском, формат `<тип>(<область>): T-XXX — <что сделано>`.
- Если pre-commit что-то изменил — запустить ещё раз, закоммитить добавленные изменения отдельным коммитом.

## 8. Протокол хендофа (КРАТКО)

Каждая сессия:
1. Читает `HANDOFF.md` → берёт первую неотмеченную задачу из `TASKS.md`.
2. Делает её (5-10 мин).
3. Коммит + push + deploy.
4. Ставит `[x]` в `TASKS.md`, обновляет `HANDOFF.md`.
5. Сообщает Pavel'у факт + ссылку.
6. `block_on_user=true`.

Полный протокол — в `START_PROMPT.md`.
