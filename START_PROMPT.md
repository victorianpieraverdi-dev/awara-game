## Что делать после того, как сессия отработала

Просто открой новую Devin-сессию на следующем аккаунте, скопируй тот же блок выше. Devin сам подтянет последний `HANDOFF.md` и продолжит со следующей `[ ]` задачи.

## Если задача оказалась с пробелом (ASK:)

Pavel отвечает на ASK прямо в чат → Devin фиксирует ответ в `HANDOFF.md` (раздел «Решения по ASK»), снимает префикс ASK с задачи, продолжает.

## Если PR долго не мерджится

Если Pavel не успел смерджить PR за день — следующий аккаунт может взять следующую задачу из TASKS.md и работать параллельно (другая feature-ветка, другой PR). Это нормально. PRs не конфликтуют, пока не трогают одни и те же строки.
